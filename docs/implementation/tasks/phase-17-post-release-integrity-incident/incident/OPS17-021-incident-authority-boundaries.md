# Task: OPS17-021 — Incident Authority Boundaries

## Purpose

Predefine incident response authority and ensure incidents do not grant new powers (Domain AH).

## Implementation Notes

- Define responder roles and scope.
- Ensure response:
  - is attributable
  - does not bypass IAM
  - does not suspend audit/accountability

## Acceptance Criteria

- Authority is predefined and documented.
- No "incident exception" exists to governance.
