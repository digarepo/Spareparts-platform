# Task: REL16-080 — Hotfix + Patch Governance

## Purpose

Operationalize post-release change governance (Domain AG).

## Implementation Notes

- Hotfix:
  - narrowly scoped corrective change
  - no new features
  - no semantic changes
- Patch release:
  - governed release bundling hotfixes
  - same governance as initial release

## Acceptance Criteria

- Hotfix/patch definitions are documented.
- Approval and audit steps exist.
