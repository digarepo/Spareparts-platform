# Task: PRISMA-41 — Define inventory and reservation persistence schema
Blueprint: Domain K (Inventory Core Model), Domain L (Inventory Availability & Reservation)
Phase: 04 Inventory

## Layer

prisma

## Package / Area

packages/db (Prisma schema)

## Purpose

Define the authoritative persistence model for inventory quantities and reservations, ensuring:
- tenant ownership is explicit and enforceable
- quantities are discrete integers and cannot become negative
- reservations are first-class, traceable records distinct from inventory on-hand

## Implementation Location

* packages/db/prisma/schema.prisma (inventory + reservation models)
* packages/db/prisma/migrations/* (generated by Prisma)

## Implementation Notes

* Inventory model must:
  * Be tenant-scoped (`tenant_id` present and indexed)
  * Reference catalog variant (`variant_id` stored as ULID string; relation may be logical, not a DB FK if cross-schema)
  * Represent quantities as integers
  * Separate quantity concepts as per blueprint:
    * on-hand
    * reserved
    * allocated
  * Enforce invariants:
    * quantities are integers
    * no negative values
    * reserved + allocated <= on_hand
* Reservation model must:
  * Have stable identity (`reservation_id` ULID)
  * Be tenant-scoped
  * Reference `variant_id`
  * Store `quantity` (positive integer)
  * Store lifecycle state (e.g. active/released/expired) if modeled
  * Store timestamps (`created_at`, `released_at` / `expires_at` as applicable)

## Acceptance Criteria

* Prisma schema includes:
  * Inventory model/table
  * InventoryReservation model/table
* Inventory table has:
  * unique constraint ensuring one inventory record per (tenant_id, variant_id)
  * indexes on tenant_id and variant_id
  * check constraints preventing negative quantities and preventing oversubscription
* Reservation table has:
  * primary key reservation_id
  * indexes on tenant_id and variant_id
* Migration can be generated/applied without errors

## Dependencies

* CONTRACTS-41 — Inventory identifier contracts (for identifier naming/shape conventions)
